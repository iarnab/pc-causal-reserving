# Explain Flag — Agent 2: Anomaly Detection

Explain a specific anomaly flag in plain English — what was detected,
why it was flagged, and what it might mean actuarially.

## Usage

```
/explain-flag <lob> grcode=<company> accident_year=<year> development_lag=<lag>
```

Example:
```
/explain-flag WC grcode=353 accident_year=1992 development_lag=3
```

---

## Step 1 — Retrieve the Flag

Query `anomaly_flags`:

```sql
SELECT *
FROM anomaly_flags
WHERE lob = ? AND grcode = ?
  AND accident_year = ? AND development_lag = ?
ORDER BY detector;
```

If no flag found: `No anomaly flag recorded for this cell. Run /scan-anomalies first.`

Show the ATA factor for the flagged cell vs peers:

## Step 4 — Plain-English Explanation

Produce a structured actuarial explanation:

```
ANOMALY EXPLANATION
═══════════════════════════════════════════════════════════
Cell:       LOB=WC, Company=353, AY=1992, Development Lag=3
Detectors:  Z-score (Z=3.14, CRITICAL), Diagonal Regression (HIGH)

What was detected:
  The age-to-age development factor from lag 2 to lag 3 for accident
  year 1992 is 2.32, compared to a peer median of 1.39. This is 3.14
  standard deviations above the mean — statistically unusual.

Potential actuarial explanations:
  1. Large late-reported claims: A major workers' comp claim (e.g.,
     occupational disease, serious injury) may have emerged late in
     the development period for this cohort.
  2. Reserve strengthening: The company may have conducted a bulk
     reserve increase for accident year 1992 at the 3-year mark.
  3. Data entry error: The value 11,847 may be a transcription error
     (cf. neighbouring cells: 7,104 and 8,532).
  4. Policy change: A change in claims handling or reporting protocol
     in 1992–1995 could have affected development patterns.

Recommended next steps:
  /trace-anomaly WC grcode=353 accident_year=1992 development_lag=3
    → trace causal pathway in the DAG

  /draft-narrative WC grcode=353
    → generate actuarial narrative with this flag as context

```

## Notes

- This command is **read-only** — no DB writes
- Explanations are generated by the agent (not the Claude API) — no API cost
- For a full causal trace, use `/trace-anomaly` after this command
